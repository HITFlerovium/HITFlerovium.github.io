从 SFML 2 迁移到 SFML 3

从 SFML 2 迁移到 SFML 3 欢迎来到 SFML 3！ SFML 团队付出了巨大的努力，旨在提供一个既能让现有用户感到熟悉，又带来了重大改进的库。 本文档将指导您如何将现有的 SFML 2 应用程序升级至 SFML 3。 SFML 3 最大的变化之一是将 C++ 标准提升到了 C++17，从而将 SFML 带入了现代 C++ 的世界！ 这一改变不仅促成了大量的内部改进，也为优化 API 提供了新的机遇，这些我们都会在本文档中逐一探讨。 编译器要求 随着标准从 C++03 升级到 C++17，您可能也需要升级编译器。 在 SFML 3 发布之前的几年里，所有主流编译器均已广泛支持 C++17，因此您目前正在使用的编译器极有可能已经支持。 如果您确实需要升级，以下是最低的编译器版本要求： 编译器 版本 MSVC 16 (VS 2019) GCC 9 Clang 9 AppleClang 12 CMake 目标 SFML 3 采用了现代 CMake 库目标的命名惯例，即在目标名称前添加一个命名空间。 这些命名空间明确告诉 CMake “这是一个目标”，而像 sfml-graphics 这样的名称，既可能是一个目标，也可能是磁盘上的一个预编译库，比如 libsfml-graphics.so。 消除这种歧义后，在因为（例如）忘记调用 find_package 而找不到某个目标时，CMake 可以提供更有用的错误信息。 此外，调用 find_package 时使用的组件名称现在也改为了首字母大写。 ...
